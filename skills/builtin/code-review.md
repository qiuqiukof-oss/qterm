# 代码审查技能（Code Review）

按以下清单逐项审查代码变更，给出可执行的修改建议（附文件:行号）：

## 1. 正确性
- 边界条件：空值、空数组、0、负数、超大输入是否处理？
- 并发/异步：竞态、未 await、闭包捕获的循环变量。
- 错误处理：异常是否被吞掉？失败路径是否回滚？

## 2. 安全性
- 注入：SQL/命令/模板注入，拼接的用户输入。
- 鉴权：敏感操作是否校验权限？越权访问？
- 密钥：是否硬编码了 token / 密码？

## 3. 性能
- N+1 查询、不必要的全量循环、未加索引的过滤。
- 大对象拷贝、内存泄漏（未释放的监听器/定时器）。

## 4. 可维护性
- 命名是否达意？重复逻辑是否应抽函数？
- 是否有注释解释「为什么」而非「是什么」？

输出格式：先给总体结论（可合并 / 需修改 / 阻塞），再按严重度列出问题清单。
