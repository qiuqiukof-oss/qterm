核心差异 · Round-Table

真正的协作，发生在 Agent 之间

大多数「浏览器里的终端」只是把 xterm 搬上网页。Hesi 的不同在于：它假设真实的工作量发生在多个 Agent 的相互质疑与补充之间，而非人与单条命令之间。

  • 多选伙伴
    从已发现的 CLI Agent 中多选（最多 4 个）作为讨论对象。
  • 多轮迭代
    AI 助手与 Agent 交替发言，逐轮收敛方案。
  • 增量抽取
    每轮只把 Agent 的新增输出喂给 AI，避免上下文重复污染。
  • 超时与清理
    单会话 5 分钟超时终止，完成会话 5 分钟 TTL 后自动清理。

The Name

命名由来 · 一个名字，两层巧思

中文

合思（Hé Sī）

  • 合 — 协作与结合：多个 Agent 汇聚一处，中西工具生态在此对接。
  • 思 — 思考与灵感：AI 的智力活动，也是那一闪而过的念头。
  • 合起来，「合思」谐音「巧合的灵思」，带一点"神来之笔"的意外之喜。两个字，过目不忘。
English

HeSi

  • HeSi — four letters, one keystroke away in your terminal. A name built for the CLI.
  • He + Si (pinyin for 「思」, thinking) reads almost exactly as "He thinks" — a thinking companion, a purely delightful coincidence.
  • Ends on a vowel, rolls right off the tongue.

Quick Start

三行命令，跑起来

本地自托管，默认端口 4264。开箱即用，无需任何外部服务。

$ git clone https://github.com/qiuqiukof-oss/Hesi.git
$ cd Hesi && npm install
$ npm start # → http://localhost:4264
✓ Hesi（合思） listening on 127.0.0.1:4264
