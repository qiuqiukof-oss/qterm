📚 如何创建插件
1
使用面板「新建」向导
在主应用右侧面板的 🔌 插件 标签页中，点击 + 新建 按钮。 填写插件名称、描述、作者信息，选择所需的功能模块，一键生成插件骨架。
2
手动创建插件目录
在 plugins/ 目录下创建以插件名命名的文件夹，包含 plugin.json 清单文件。 最少只需一个 plugin.json 即可注册： 
{
  "name": "my-plugin",
  "version": "0.1.0",
  "description": "我的第一个插件",
  "author": "My Name",
  "license": "MIT"
}
3
添加功能模块
在 plugin.json 中添加能力声明，插件支持以下 7 种能力：
🔧
CLI 工具
注册自定义 CLI，在终端中使用。需提供可执行路径。
⚡
工作流
定义多步骤自动化工作流，支持顺序执行和条件判断。
🤖
AI 工具
为 AI 助手注册自定义工具，扩展对话能力。
🌐
HTTP 路由
注册自定义 API 端点，处理 HTTP 请求。
🖥️
UI 面板
注册右侧面板标签页，渲染自定义 HTML/JS 内容。
🔗
MCP 服务
注册 MCP 服务器，通过 Model Context Protocol 提供工具。
🔄
生命周期
定义加载/卸载钩子，在插件生命周期中执行初始化和清理。
4
分享你的插件

创建好插件后，将其发布到 GitHub 并添加 cli-q-plugin 话题标签，即可自动出现在社区发现中：

  1. 在 GitHub 创建仓库并推送 plugin.json
  2. 进入仓库 Settings → Topics，添加 cli-q-plugin 标签
  3. 插件将在 5 分钟 内自动出现在社区发现中
  4. 也可以将 npm publish 并添加 cli-q-plugin 关键词
5
详细文档
完整的插件开发规范请参阅项目中的 PLUGIN_SPEC.md 文件，内含：
  • 完整的 plugin.json 字段说明
  • 每种能力的详细配置示例
  • 生命周期事件参考
  • 插件市场发布指南
